---
title: API Tokens
description: "Manage API tokens from the CLI"
---

API tokens authenticate SDK and API requests without requiring a Supabase JWT. Tokens start with `wh_` and never expire (until revoked).

## Create a token

```bash
wago tokens create TOKEN_NAME
```

```
Created API token: my-app

  wh_your_api_token_here

  Save this token — it won't be shown again.
```

<Callout type="warn">
  The raw token is shown only once at creation time. Wago stores a SHA-256
  hash — the original token cannot be retrieved.
</Callout>

## List tokens

```bash
wago tokens list
```

```
ID                                    NAME                  PREFIX           LAST USED             CREATED
────────────────────────────────────  ────────────────────  ───────────────  ────────────────────  ────────────────────
ef67dace-c3e2-43df-a5ad-18ba65395eb1  my-app                wh_12c6edf...    2026-03-14T04:12:00   2026-03-14T04:12:13

1 token(s)
```

## Revoke a token

```bash
wago tokens revoke TOKEN_ID
```

```
Token revoked
```

Revoked tokens immediately stop working. Any SDK or API client using the token will get a 401 error.

## Aliases

```bash
wago t list
wago t create my-token
wago t revoke TOKEN_ID
```
