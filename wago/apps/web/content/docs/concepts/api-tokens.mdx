title: API Tokens

API tokens provide a simple way to authenticate with the Wago API from your application, scripts, or CI/CD pipelines.

## Token format

Tokens follow the format `wh_` + 48 hex characters:

```
wh_your_api_token_here
```

## Security

- Tokens are **hashed with SHA-256** before storage — Wago never stores the raw token
- The raw token is shown **only once** at creation time
- Tokens **never expire** but can be revoked at any time
- Each API request with a token updates the `lastUsedAt` timestamp

## Usage

Pass the token as a Bearer token in the `Authorization` header:

```bash
curl https://api.wago.com/api/connections \
  -H "Authorization: Bearer wh_..."
```

Or use an SDK:

<Tabs items={["TypeScript", "Python"]}>
  <Tab value="TypeScript">
```typescript
const client = new Wago({ apiKey: 'wh_...' });
```
  </Tab>
  <Tab value="Python">
```python
client = Wago(api_key="wh_...")
```
  </Tab>
</Tabs>

## Token vs JWT

The Wago API accepts two types of authentication:

| | API Token (`wh_...`) | Supabase JWT |
|---|---|---|
| Lifetime | Until revoked | 1 hour |
| Use case | SDKs, scripts, CI/CD | Dashboard, CLI |
| Creation | API or CLI | Login flow |
| Refresh | Not needed | Automatic |

## Best practices

- Create **separate tokens** for each application or environment
- **Revoke tokens** immediately if they're compromised
- **Never commit tokens** to version control — use environment variables
- Use descriptive names so you know which token is used where
